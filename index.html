Codeigniter Matches CLI

Codeigniter Matches is a cli script that allows you to write controllers, models and views faster.

Usage

  1. To verify if you have php cli installed, type into terminal/command prompt.
    php -v
  2. If you receive the php version number, then you are good to go.
  3. Go to your config.php file inside the application/config and make sure the following line looks like below:
    $config['uri_protocol'] = 'AUTO';
  4. After that line, add:
    $config['uri_protocol'] = isset($_SERVER['REQUEST_URI']) ? 'PATH_INFO' : 'CLI';
  5. Save the file...
  6. Put Matches.php (or matches.php if you have a lesser than v.3 Codeigniter) inside the controllers directory. Put config/matches.php inside your app config folder. Also, put matches_templates folder inside the views folder.
  7. From terminal or command prompt go to the application's index.php and type:
    php index.php matches
  8. If everything went well, you should be greeted by Matches.

Commands

You can start using Matches by writing:
php index.php matches [COMMAND]

...where [COMMAND] can be:

create

  • create app name_of_app - creates a MVC stack (controller, model, view) having name_of_app as names.
  • create controller name_of_controller - creates a controller having name_of_controller as name.
  • create migration name_of_migration name_of_table-(OPTIONAL) - creates a migration having name_of_migration prefixed with version as file name. If no name_of_table is given, will name the table as the name_of_migration.
  • create model name_of_model - creates a model having name_of_model as name.
  • create view name_of_view - creates a view having name_of_view as file name.

encryption_key

  • encryption_key string_to_hash-(OPTIONAL) - creates an encryption key inside all config.php's found in config folder. If $config['encryption_key'] = ''; doesn't exist or has a value, the encryption key won't be written.
